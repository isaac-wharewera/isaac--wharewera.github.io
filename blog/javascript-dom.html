Explain control flow and loops using an example process from everyday life.

  • Control Flow: The technical term for specifying the sequence in which a code executes. Conflow Flow is like the sequential steps you have to follow on a recipe when baking a cake.
  • Loops: Used to repeat a block of code. Offers a quick and easy way to do SOMETHING repeatedly. An example of this would be if you set out to preform 1 push-up that would be considered the SOMETHING we did. We can use loops to repeat that SOMETHING 100 times, effectively making us do 100 push-ups.

Describe what the DOM is and an example of how you might interact with it.

The DOM is a tree-like representation of the web page that gets loaded into the browser. It represents the web page using a series of objects. The main object is the document object, which in turn houses other objects which also house their own objects, and so on. Those objects are usually refered to as nodes. A way to interact with it is by using the JavaScript API to modify or change the HTML document.

Explain the difference between accessing data from arrays and objects.

  • To access data from arrays you must either access it through its binding i value(value[i]) or with its own value or name. An example if we had the array [2, 3, 7, 4, 9] to access the fist data in the array we would use value[0] or value["2"].
  • To access objects we use something called "dot property accessor" which is a common way to access the properties of an object. An example is if we have a object with the value (tree) and a property with the value( branches: 'big'). We can access this property by using "dot property accessor" which looks like this: tree.branches.

Explain what functions are and why they are helpful.

Functions are a block of code designed to perform a particular task. A JavaScript function is executed when "something" invokes it (calls it). They are helpful because functions allow you to define a block of code, give it a name and then execute it as many times as you want. So if you create an Element that later needs to perform an already existing function instead of duplicating or re-writting code you can just call the function.
